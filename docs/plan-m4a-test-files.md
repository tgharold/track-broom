# Plan: M4A Test File Support

## Goal
Create M4A test fixtures and utilities following the same pattern as existing MP3 test files (`test_mp3s/`, `tests/utils.encode_mp3`, `tests/utils.add_id3_tags`).

## Background
- `test_mp3s/` contains 3 MP3 tone files generated by `scripts/generate_test_tones.py`
- `tests/utils/__init__.py` provides `tone_samples`, `encode_mp3`, `add_id3_tags`
- `tags.py` already supports M4A (81 lines of M4A handling in `tags.py`) but has **no dedicated tests**
- M4A fixtures in `tests/test_scanner.py` are created with `.touch()` (empty 0-byte files)

## Changes

### 1. `tests/utils/__init__.py` ‚Äî M4A utility functions

Add two new functions:

- `encode_m4a(pcm_data, output_path, samplerate, channels) -> str`
  - Pipe raw PCM to ffmpeg via `pipe:0`
  - Encode with ALAC codec into M4A container
  - Same pattern as `encode_mp3`, just different codec/output format

- `add_m4a_tags(filepath, title, artist, album, genre, tracknumber) -> None`
  - Write tags using `mutagen.mp4.MP4`
  - Use Apple/MP4 field keys: `nam`, `ART`, `alb`, `day`, `gnr `, `trkn`, `disk`
  - Same API shape as `add_id3_tags` for consistency

`tone_samples` is already shared (raw PCM output, no encoding dependency).

### 2. `scripts/generate_test_tones_m4a.py` ‚Äî M4A generation script

Mirrors `scripts/generate_test_tones.py`:

- Output to `test_m4as/` directory
- Same tones as MP3s (note_C, note_A, note_H) with same metadata
- Uses `encode_m4a` and `add_m4a_tags` instead of MP3 equivalents

### 3. `tests/test_m4a.py` ‚Äî New test file

Mimics `tests/test_generate_tone.py` structure:

**TestEncodeM4a** (4 tests):
- `test_file_created` ‚Äî check file exists and non-zero
- `test_custom_sample_rate` ‚Äî stereo with different sample rate
- `test_invalid_pcm_raises_error` ‚Äî garbage PCM raises ffmpeg.Error
- `test_different_bitrate` ‚Äî encode with low bitrate

**TestAddM4aTags** (3 tests):
- `test_tags_written` ‚Äî write all tags, reload with `mutagen.mp4.MP4`, verify all 5
- `test_title_only` ‚Äî only title required, others omitted
- `test_tracknumber_default` ‚Äî default tracknumber of 1

### 4. `test_scanner.py` ‚Äî Replace `.touch()` M4A fixture

Current: `(sub2 / "hidden.m4a").touch()` creates an empty 0-byte file.

Should: Use `encode_m4a` + `add_m4a_tags` to create a real M4A file
- This alone would increase the file count and change test expectations
- More importantly, enables future tag-reading tests

### 5. Updated test count: 28 ‚Üí **38 tests**
- `test_generate_tone.py`: 14 tests (unchanged)
- `test_m4a.py`: 7 tests (new)
- `test_scanner.py`: 13 tests (unchanged)

### 6. Dependencies (none)
- ffmpeg already has ALAC encoder available (`ffmpeg -encoders | grep alac`)
- mutagen's `MP4` class is already used by `tags.py`
- No new pip dependencies needed
